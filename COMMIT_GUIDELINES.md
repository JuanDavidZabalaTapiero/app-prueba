# Convenciones de commits

Objetivo: mantener mensajes claros y consistentes.

Formato recomendado:
tipo: descripción breve

Tipos comunes:
- feat: nueva funcionalidad
- fix: corrección de bug
- docs: documentación
- style: formato o apariencia
- refactor: reestructuración sin cambiar comportamiento
- chore: tareas varias

Ejemplos:
- feat: añadir formulario de contacto
- docs: actualizar README.md
- fix: corregir cierre de etiqueta en header

Idioma: español.
